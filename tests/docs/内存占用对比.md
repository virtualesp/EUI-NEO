# 内存占用对比

本文记录当前构建在同一台 macOS 机器上的内存占用，并和 2026-06-04 的旧采样结果做同口径对比。

## 环境

| 项目 | 旧采样 | 当前采样 |
| --- | --- | --- |
| 时间 | 2026-06-04 11:18 CST | 2026-06-09 13:08 CST |
| 系统 | macOS 26.5，Darwin 25.5.0，arm64 | macOS 26.5.1，Darwin 25.5.0，arm64 |
| CPU | Apple M5 | Apple M5 |
| 构建目录 | `build/probes-opengl-glfw` | `build/probes-opengl-glfw` |
| 构建配置 | `Release`、`EUI_RENDER_BACKEND=opengl`、`EUI_WINDOW_BACKEND=glfw`、`EUI_BUILD_TEST_FIXTURES=ON`、`EUI_DEPS_MODE=bundled` | 同左 |
| 采样方法 | 进程启动后等待 3 秒，使用 `ps -o rss= -o %cpu=` 每秒采样 6 次；同时使用 `vmmap -summary` 记录内存构成 | 同左 |

## 测试目标

| 目标 | 窗口尺寸 | 含义 |
| --- | --- | --- |
| `window_only_probe` | 800x600 | 只初始化 GLFW 并创建 OpenGL-capable 窗口，不创建 EUI RenderBackend。 |
| `render_context_probe` | 800x600 | 创建窗口并初始化 EUI RenderBackend / OpenGL context，不执行 clear/present。 |

| `animation_240fps` | 1280x720 | 真实 DSL 动画压力场景，目标 240 FPS。 |

## 与 2026-06-04 对比

| 目标 | RSS | Physical footprint | Peak footprint | MALLOC resident | MALLOC allocated | 主要变化 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| `window_only_probe` | -0.4 MiB | -0.4 MiB | -0.5 MiB | -0.1 MiB | -0.1 MiB | 无明显单项变化 |
| `render_context_probe` | +0.0 MiB | -0.1 MiB | +2.1 MiB | +0.1 MiB | +0.0 MiB | 无明显单项变化 |
| `render_clear_probe` | +0.0 MiB | +0.0 MiB | +0.0 MiB | +0.3 MiB | -0.1 MiB | 无明显单项变化 |
| `empty_window` | +0.1 MiB | -0.1 MiB | +2.2 MiB | +0.3 MiB | -0.1 MiB | 无明显单项变化 |
| `animation_240fps` | +0.4 MiB | +7.5 MiB | +0.2 MiB | +0.6 MiB | +0.1 MiB | `IOAccelerator (graphics)` +2.6 MiB，`owned unmapped graphics` +0.8 MiB |

## 判断

- 这轮基础目标没有变大：`window_only_probe`、`render_context_probe`、`render_clear_probe` 和 `empty_window` 的 RSS / Physical footprint 基本都在旧采样误差范围内。
- `empty_window` 的 `MALLOC allocated` 仍约 6.6 MiB，说明当前 DSL Runtime 空页面没有因为 Loader / `ui.state` 改动出现普通 heap 暴涨。
- `animation_240fps` 的 `Physical footprint` 从 164.7 MiB 升到 172.2 MiB，但 RSS 只升 0.4 MiB，`MALLOC allocated` 只升 0.1 MiB。主要增量不是 C++ 对象或组件实例状态，而是动态图形场景触发的 driver / graphics residency。
- 当前源码引入 MD4C 和 Loader 相关改动后，基础 app 的常驻内存并没有显著抬高；如果只看 Activity Monitor 或 `ps` RSS，会把系统 framework、窗口系统、OpenGL/IOSurface 常驻页一起算进去，容易误判。
