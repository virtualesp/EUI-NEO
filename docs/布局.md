# 布局系统

布局系统位于 `core/layout.h`，是 header-only 的纯计算模块，不依赖 OpenGL。DSL 中的 `Row / Column / Stack` 最终都会转换成 `core::Node` 进行 measure 和 layout。

## 容器类型

```cpp
enum class LayoutType {
    Row,
    Column,
    Stack,
    Flow
};
```

- `Row`：子元素横向排列。
- `Column`：子元素纵向排列。
- `Stack`：子元素叠放。
- `Flow`：子元素横向流式排列，放不下时自动换行。

DSL 写法：

```cpp
ui.row("toolbar")
ui.column("content")
ui.stack("root")
ui.flow("chip.group")
```

## 对齐

```cpp
enum class Align {
    START,
    CENTER,
    END
};
```

每个布局节点有：

- `mainAlign`：主轴对齐。
- `crossAlign`：交叉轴对齐。

规则：

- `Row` 主轴是 x，`crossAlign` 控制 y。
- `Column` 主轴是 y，`crossAlign` 控制 x。
- `Stack` 使用 `mainAlign` 控制 y，使用 `crossAlign` 控制 x。

DSL 写法：

```cpp
.justifyContent(core::Align::CENTER)
.alignItems(core::Align::CENTER)
.align(core::Align::CENTER, core::Align::CENTER)
```

## 尺寸模式

```cpp
enum class SizeMode {
    Fixed,
    WrapContent,
    Fill
};
```

`SizeValue`：

```cpp
SizeValue::fixed(240.0f)
SizeValue::wrapContent()
SizeValue::fill()
```

DSL 写法：

```cpp
.width(240.0f)
.height(70.0f)
.size(240.0f, 70.0f)
.width(core::SizeValue::fill())
.height(core::SizeValue::wrapContent())
.fill()
.wrapContent()
```

当前 `Fill` 已支持基础的剩余空间分配，并可结合 `flexGrow / flexShrink` 使用，但它还是轻量实现，不等同于完整 CSS Flexbox。

## 位置

布局位置是逻辑坐标，原点在左上角，y 向下。

```cpp
.x(40.0f)
.y(24.0f)
.position(40.0f, 24.0f)
```

在 `Stack` 中，指定 x/y 会覆盖对应方向的自动对齐；未指定时按 `align` 规则放置。

在 `Row / Column` 中，子项位置主要由布局流决定，x/y 当前仍会进入 layout node，但推荐把显式定位主要用于 `Stack`。

## Margin 与 Gap

Margin：

```cpp
.margin(8.0f)
.margin(12.0f, 8.0f)
.margin(12.0f, 8.0f, 12.0f, 8.0f)
```

Gap：

```cpp
.gap(12.0f)
.spacing(12.0f)
```

`gap/spacing` 对 `Row`、`Column` 和 `Flow` 的子元素间距有意义。

`Flow` 同样使用 `gap/spacing` 控制同一行内间距，并用 `lineGap/lineSpacing` 控制换行后的行距：

```cpp
.gap(12.0f)
.lineGap(10.0f)
```

## Padding

容器现在支持内边距，会参与 measure 和 layout。

```cpp
.padding(12.0f)
.padding(16.0f, 12.0f)
.padding(20.0f, 14.0f, 20.0f, 14.0f)
```

优先用 `padding` 表达容器内部留白，而不是额外套一层仅用于缩进的空容器。

## Min / Max 尺寸

布局节点现在支持最小/最大尺寸约束：

```cpp
.minWidth(180.0f)
.minHeight(44.0f)
.minSize(180.0f, 44.0f)

.maxWidth(420.0f)
.maxHeight(280.0f)
.maxSize(420.0f, 280.0f)
```

这在响应式卡片、按钮组和表单区域里比手动 `std::min/std::max` 计算更稳定。

## Flex

`Row` / `Column` 主轴上支持基础 flex 分配：

```cpp
.width(core::SizeValue::fill())
.flexGrow(1.0f)
.flexShrink(1.0f)

.flex(1.0f, 1.0f)
```

规则：

- 主轴上 `Fill` 或显式 `flexGrow > 0` 的子项会参与剩余空间分配。
- `flexShrink > 0` 的子项在空间不足时会参与压缩。
- 这是轻量分配模型，适合常见等比分栏、自适应表单和控制组，不包含完整 CSS flex 的 basis / wrap / order 等语义。

## Measure / Layout 流程

Runtime 每次 compose 后会调用：

```cpp
ui.layout(screen);
```

内部流程：

1. 把 DSL `Element` 树转换为 `core::Node` 树。
2. 调用 `measure(availableWidth, availableHeight)` 计算尺寸。
3. 调用 `layout(x, y)` 计算最终 frame。
4. 把每个 `Node::frame()` 写回对应 `Element::frame`。

最终 frame 存在逻辑坐标中，渲染阶段再乘 `dpiScale` 转成 framebuffer 像素。

## Row 示例

```cpp
ui.row("toolbar")
    .size(420.0f, 64.0f)
    .gap(12.0f)
    .alignItems(core::Align::CENTER)
    .content([&] {
        components::button(ui, "save")
            .size(120.0f, 44.0f)
            .text("Save")
            .build();

        components::button(ui, "cancel")
            .size(120.0f, 44.0f)
            .text("Cancel")
            .build();
    });
```

Row 会：

1. 计算所有子元素 outer width 总和。
2. 加上 gap。
3. 根据 `justifyContent` 决定 x 起点。
4. 根据 `alignItems` 决定每个子元素 y。

## Column 示例

```cpp
ui.column("panel.content")
    .size(360.0f, 260.0f)
    .gap(10.0f)
    .justifyContent(core::Align::CENTER)
    .alignItems(core::Align::CENTER)
    .content([&] {
        ui.text("title")
            .size(300.0f, 42.0f)
            .text("Gallery")
            .fontSize(32.0f)
            .build();

        components::button(ui, "primary")
            .size(220.0f, 58.0f)
            .text("Open")
            .build();
    });
```

Column 会：

1. 计算所有子元素 outer height 总和。
2. 加上 gap。
3. 根据 `justifyContent` 决定 y 起点。
4. 根据 `alignItems` 决定每个子元素 x。

## Stack 示例

```cpp
ui.stack("stage")
    .size(600.0f, 320.0f)
    .content([&] {
        ui.rect("stage.bg")
            .size(600.0f, 320.0f)
            .color({0.10f, 0.12f, 0.16f, 1.0f})
            .build();

        ui.rect("actor")
            .x(active ? 420.0f : 40.0f)
            .y(80.0f)
            .size(120.0f, 80.0f)
            .transition(0.3f)
            .build();
    });
```

Stack 会按声明顺序绘制，后声明的元素盖在前面的元素上。

## Flow 示例

```cpp
ui.flow("toolbar.actions")
    .width(520.0f)
    .height(core::SizeValue::wrapContent())
    .gap(12.0f)
    .lineGap(10.0f)
    .content([&] {
        components::button(ui, "save")
            .size(140.0f, 44.0f)
            .text("Save")
            .build();

        components::button(ui, "export")
            .size(160.0f, 44.0f)
            .text("Export CSV")
            .build();

        components::button(ui, "share")
            .size(132.0f, 44.0f)
            .text("Share")
            .build();
    })
    .build();
```

`Flow` 适合按钮组、标签组、属性卡片、筛选项等“横向优先，但需要自动换行”的内容。

## 与脏区渲染的关系

布局系统只计算逻辑 frame。Runtime 会用 frame 和视觉属性计算 dirty rect：

- frame 变化会标记旧区域和新区域的 union。
- shadow / blur 会扩大脏区。
- text frame / color / opacity / transform 变化会标记文本区域。

因此写 DSL 页面时不需要手动标记脏区。

## 当前限制

- `Flow` 当前是基础流式换行容器，只处理从左到右换行，不支持按行对齐策略或瀑布流能力。
- `flexGrow / flexShrink` 已可用，但仍是轻量实现，不包含完整 CSS flexbox 语义。
- `clip` 仍是基础矩形裁剪；更完整的滚动容器建议优先用 `components::scrollView`。
- `z-index` 只影响同级绘制和 topmost hit-test，不参与布局。
- transform 后的视觉位置不会反向改变布局占位和命中矩形。
