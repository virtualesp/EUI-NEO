# EUI-NEO 稳定版路线图

本文只记录稳定版前仍需推进的任务。已完成任务和暂缓任务不再放在路线图主体里，避免后续开发继续围着旧状态转。

当前稳定版前剩余主线：

1. Vulkan 前置分层
2. Vulkan 后端支持
3. 修复 bug 到稳定版

## 架构约束

稳定版前必须收束到清晰分层，避免继续用临时适配把 GLFW、SDL2、OpenGL、Vulkan 和平台原生能力缠在一起。

- 渲染后端和窗口后端的最终边界以 [渲染后端架构](渲染后端架构.md) 为准；后续代码改动必须先对应到文档里的阶段。
- 对外 app/DSL 层不得暴露 `GLFWwindow*`、`SDL_Window*`。内部统一使用 `core::window::Handle` 或后续 `WindowId`。
- CMake 只选择窗口后端和渲染后端；不长期维护两套重复主循环。
- `WindowBackend` 只负责窗口、事件、输入法、剪贴板、光标、图标、native handle 和父子窗口关系，不负责具体绘制。
- `RenderBackend` 单独负责 OpenGL/Vulkan surface、frame lifecycle、resize、present、render cache 和 primitive 渲染，不塞进 `WindowBackend`。
- Vulkan 后端的目标是与 OpenGL 后端同一 DSL 输出画面一致；禁止用 bounding box、纯色近似、功能占位或“先能看到东西”的临时绘制替代真实 primitive/text/image 语义。
- GLFW/SDL2 事件必须先翻译为 EUI 自有事件，再进入 runtime；runtime/组件层不得出现后端键码和窗口库类型。
- 子窗口、modal、父子层级和多窗口更新归统一 WindowManager/AppRunner 管理，后端只提供底层窗口操作。
- macOS/Windows/Linux 原生能力集中到 native bridge，避免平台特例散落在窗口和渲染主循环里。
- Vulkan 开始前必须先完成窗口后端与渲染后端边界，不能把 Vulkan 特例散落在 GLFW/SDL2 main 里。
- 架构收紧时不得留下旧路径兼容壳或无用代码；迁移后要同步删除旧文件、旧 include、旧文档路径和未使用 CMake source。
- 每轮整理必须用 `rg` 扫旧路径/旧命名残留，并跑 `git diff --check` 与对应后端构建验证。

## 1. Vulkan 前置分层

状态：进行中。

目标：先让当前 OpenGL 走清晰的渲染后端接口，再接 Vulkan。不能直接新增一个 Vulkan main，把 Vulkan surface、swapchain、resize、present 和资源释放逻辑塞进 GLFW/SDL2 事件泵。

### 必须先完成

- 已完成：`core/render/render_backend.h` 改成纯接口，不包含 `<glad/glad.h>`、Vulkan 头或任何具体后端类型。
- 已完成：当前 `OpenGLRenderBackend` 从公共接口头移到 OpenGL 专属实现文件。
- 已完成：`core/dsl_runtime.h` 的 clear、viewport、scissor、render cache、framebuffer/blit 已交给 `RenderBackend` 和 OpenGL 专属实现。
- 已完成前置：`Color`、`Rect`、`Transform` 等后端无关渲染类型已拆到 `core/render/render_types.h`，不再定义在 OpenGL primitive 实现头里。
- 已完成前置：`TextStyle`、`HorizontalAlign`、`VerticalAlign` 已拆到 `core/render/text_types.h`，公共类型头不再为文本样式拉入完整 OpenGL text primitive。
- 已完成前置：`ImageFit` 已拆到 `core/render/image_types.h`，DSL builder 不再为了图片 fit 枚举直接拉入 OpenGL image primitive。
- 已完成前置：public `eui/image.h` 改为 `eui::image` 查询函数 facade，不再向用户侧暴露 `ImagePrimitive` 实现类型。
- 已完成前置：`core/dsl.h` 已改为只依赖后端无关 render/text/image 类型，具体 primitive/text/image 实现由 runtime 或组件显式 include。
- 已完成前置：theme、animation、event、text、image 头文件已改为直接依赖后端无关 render types，减少 `primitive.h` 的传播。
- 已完成前置：`core/render/primitive.h` 已只保留后端无关 primitive API；OpenGL 的 VAO/VBO/shader 逻辑已移到 `core/render/opengl/opengl_primitives.cpp`。
- 已完成前置：primitive transform、shadow 扩张、圆角矩形顶点和 polygon triangle fan 已抽到共享 `core/render/primitive_geometry.h`，OpenGL/Vulkan 不再各自复制这部分 CPU 几何语义。
- 已完成前置：`RoundedRectPrimitive` / `PolygonPrimitive` 类状态和 render dispatch 已收束到共享 `core/render/primitive.cpp`，OpenGL/Vulkan 不再各自实现一份 primitive 类。
- 已完成前置：primitive 绘制已通过 `RenderBackend::drawRoundedRect(...)` 下发后端无关 draw command；旧的 Vulkan 专属 `vulkan_primitives.cpp` 已删除。
- 已完成前置：`core/render/text.h` 和 `core/render/image.h` 不再包含 `<glad/glad.h>`，也不再向头文件暴露 `GLuint`/`GLint`/`GLenum`。
- 已完成前置：当前 text/image 的 OpenGL 实现文件已移到 `core/render/opengl/opengl_text.cpp` 和 `core/render/opengl/opengl_image.cpp`，根目录只保留公共 facade/类型入口。
- 已完成前置：`core/render/image.h` 已改为 PImpl，图片 texture、shader、VAO/VBO、GIF 状态和缓存引用不再暴露在公共 image primitive 头文件里。
- 已完成前置：`core/render/text.h` 已改为 PImpl，字体状态、layout cache、glyph cache、atlas/shader/VAO/VBO 不再暴露在公共 text primitive 头文件里。
- 已完成前置：窗口 handle/context/cursor 类型已拆到 `core/window/window_types.h`，公共 `eui/window.h` 不再拉入完整窗口后端函数。
- 已完成前置：窗口后端文件已从 `core/platform` 收束到 `core/window`；输入类型与输入队列已拆到 `core/input/input_types.h` 和 `core/input/input_state.h`，公共类型头不再拉入 GLFW/SDL2 回调层。
- 已完成前置：阶段 2 的后端无关类型已落地：`WindowCreateRequest`、`NativeWindowInfo`、`RenderApi` 和 `RenderSurface`。
- 已完成前置：阶段 3 已落地，GLFW/SDL2 窗口创建入口改为 `core::window::createWindow(WindowCreateRequest)`；OpenGL/Vulkan 所需 window hint/flag 集中在窗口后端。
- 已完成前置：阶段 4 第一段已落地，`OpenGLRenderBackend` 拥有 OpenGL context lifecycle、glad loading、make current、present 和 render cache；GLFW/SDL2 app main 不再直接 include glad、调用 `gladLoadGLLoader` 或保存 `SDL_GLContext`。
- 已完成前置：GLFW/SDL2 app main 已通过 `core::render::createRenderBackend(...)` 创建渲染后端，并通过 `core::render::windowRenderApi()` 填充窗口创建请求，不再直接依赖具体 OpenGL 后端类。
- 已完成前置：`RenderBackend::beginFrame` 已改为接收 `RenderSurface`，frame lifecycle 不再传散的 framebuffer/dpi 参数。
- 已完成前置：窗口后端已提供 `core::window::nativeWindowInfo(window)`，主窗口和子窗口的 `RenderSurface::native` 已填充，SDL2 main 不再自己读取 native handle。
- 已完成前置：`EUI_RENDER_BACKEND` 已支持 `auto|opengl|vulkan`，默认 `auto`；实验 Vulkan 未启用或不可用时会检测 SDK 但回退 OpenGL，强制 `vulkan` 会明确报错。
- 已完成前置：`stb_image` 实现已从 OpenGL image 后端拆到共享 `core/render/stb_image_impl.cpp`，窗口图标和后续 Vulkan image 不再依赖 OpenGL image 翻译单元提供解码符号。
- 已完成实验入口：`EUI_ENABLE_EXPERIMENTAL_VULKAN_BACKEND=ON` 时可编译当前 Vulkan 后端；未启用时 `auto` 继续回退 OpenGL。
- 已完成实验最小后端：当前 Vulkan 后端已落地 instance、surface、device、swapchain、render pass、framebuffer、command buffer、clear、present 和 cleanup；GLFW/SDL2 Vulkan gallery 已验证可编译。
- 已完成实验 primitive 第一段：Vulkan 已接入预编译 SPIR-V primitive pipeline；`RoundedRectPrimitive` 已按 SDF 路径支持 fill、gradient、corner radius、border、shadow、opacity、transform、clip/scissor 和 backdrop blur；`PolygonPrimitive` 已按实际 polygon 三角化绘制 fill/opacity/transform/clip。
- 已完成收束：OpenGL/Vulkan primitive 已共用 `primitive.cpp` / `primitive_geometry.h` 的 CPU 状态和几何语义，primitive draw command 已收到 `RenderBackend` 边界。
- 已完成验收入口：新增 `examples/primitive_parity.cpp`，只绘制 rect/polygon，不混入 text/image，用于 OpenGL/Vulkan primitive 截图对比；其中包含 controlled blur 样本。
- 未完成：Vulkan primitive 仍缺截图对比验收等和 OpenGL 完全一致所需的确认工作；Vulkan text/image 当前仍是 no-op 占位。
- `core/render/text.*` 拆分为公共字体排版/字形布局和后端 GPU atlas/draw。公共层不得持有 `GLuint`。
- `core/render/image.*` 拆分为公共图片解码/GIF/远程缓存和后端 texture/draw。公共层不得持有 `GLuint`。
- `core/app/glfw_app_main.cpp` 和 `core/app/sdl2_app_main.cpp` 已只选择并创建渲染后端；下一步做 Vulkan primitive 截图对比验收，再把 Vulkan text/image 的 no-op 占位替换为真实后端实现。
- 已完成第一阶段：`CMakeLists.txt` 增加 `EUI_RENDER_BACKEND=auto|opengl|vulkan`，默认 `auto`；只有 OpenGL resolved 后端才链接 `glad` 和 `OpenGL::GL`。当前 Vulkan 仍需显式实验开关，不进入稳定默认路径。

### 阶段验证

- GLFW 默认 OpenGL 构建必须保持可用。
- SDL2 OpenGL 构建必须保持可用。
- public facade 和 examples 不得感知 OpenGL/Vulkan 后端。
- `rg` 扫描 public include 和 examples，不应出现 `glad`、`OpenGL`、`Vulkan`、`GLFWwindow*`、`SDL_Window*`。

## 2. Vulkan 后端支持

状态：实验最小后端进行中。

目标：在不破坏 GLFW/SDL2 窗口后端边界的前提下增加 Vulkan 渲染后端。

### 接入顺序

- 已完成实验：增加 Vulkan backend，先完成 instance/device/surface/swapchain/frame lifecycle。
- 已完成实验：自动选择策略固定为：Vulkan SDK 存在且实验 Vulkan backend 显式启用时使用 Vulkan，否则回退 OpenGL；强制 `vulkan` 不允许静默回退。
- 已完成：GLFW Vulkan 路径窗口创建使用 `GLFW_CLIENT_API = GLFW_NO_API`。
- 已完成：SDL2 Vulkan 路径窗口创建使用 `SDL_WINDOW_VULKAN`。
- 已完成：窗口后端只提供 native handle 或 surface 创建所需信息，不直接参与 Vulkan draw/present。
- 已完成实验：首轮 Vulkan 只做 clear color + present + resize + resource cleanup。
- 第二轮接 primitive：已完成真实 pipeline 第一段，并抽出共享 CPU 状态、几何语义和 backend draw command；`primitive_parity` 已作为独立验收入口，Vulkan backdrop blur 已走真实 swapchain snapshot + sampler 路径。下一步做截图/人工对比和剩余视觉差异，不接受 bounding box 或纯色近似。
- 第三轮接 text atlas。
- 第四轮接 image texture/GIF/remote image。
- 最后接 render cache、dirty rect 和 blit/resolve 策略。

### 必须定义清楚

- surface 创建和销毁归属。
- swapchain resize 和 minimized/zero-size 窗口行为。
- frame acquire、command record、submit、present 的失败恢复策略。
- texture、atlas、pipeline、descriptor、buffer 的释放时机。
- GLFW/SDL2 多窗口和 modal 子窗口的资源隔离策略。

## 3. 修复 bug 到稳定版

状态：持续进行。

要求：

- 每个阶段都回归 GLFW 默认构建。
- SDL2 后端在窗口、IME、modal 子窗口、托盘、焦点、多窗口行为上保持手测通过。
- 发布前构建验证 `gallery`、`demo`、`clock`、`calculator`、`serial_tool`。
- 最后阶段冻结功能，只修复 bug、补文档和做发布检查。
