# 开发与发布

本文记录 EUI-NEO 的本地开发、依赖模式、构建验证和 release 约定。目标是让每次改动都能被快速复现、审查和发布。

## 依赖模式

项目通过 `EUI_DEPS_MODE` 控制构建期依赖来源：

- `auto`：默认模式。优先使用 `3rd/` 下的本地源码，缺失时才从固定上游地址联网拉取。
- `bundled`：严格离线模式。只使用 `3rd/` 下的源码，缺失依赖直接报错。
- `fetch`：强制联网模式。从固定上游地址拉取构建期依赖。

HarfBuzz shaping 默认启用，可用 `-DEUI_ENABLE_HARFBUZZ=OFF` 关闭。

内置和 fetch 下载的第三方依赖默认按静态链接构建，包括 GLFW、glad、FreeType、HarfBuzz、libpng 和 zlib。Release 包不再额外收集 GLFW DLL / dylib / so。只有选择系统 SDL2 包时，SDL2 仍可能按系统包提供的方式动态链接。

## 渲染后端选择

项目通过 `EUI_RENDER_BACKEND` 控制渲染后端：

- `opengl`：默认模式，使用 OpenGL。
- `auto`：自动模式。Vulkan SDK 存在时选择 Vulkan，否则回退 OpenGL。
- `vulkan`：强制 Vulkan；如果 SDK 不存在，配置阶段直接报错。

Vulkan 后端已作为正式渲染后端接入。需要测试 Vulkan 时使用：

```bash
cmake -S . -B build-vk
cmake --build build-vk --target gallery --parallel
```

默认构建不需要额外配置文件，也不会因为机器装了 Vulkan SDK 自动切到 Vulkan。

构建目录后缀会在首次配置时自动选择常见组合：

- `build`：GLFW + OpenGL。
- `build-sdl2`：SDL2 + OpenGL。
- `build-vk` / `build-vulkan`：GLFW + Vulkan。
- `build-sdl2-vk` / `build-sdl2-vulkan`：SDL2 + Vulkan。

已有构建目录的 CMake cache 优先级更高；切换后缀后如果没有生效，删除旧 build 目录，或显式传入 `-DEUI_WINDOW_BACKEND=...` / `-DEUI_RENDER_BACKEND=...`。

显式 OpenGL / Vulkan 对照测试推荐使用独立构建目录：

```bash
cmake -S . -B build
cmake --build build --target gallery --parallel
cmake -S . -B build-vk
cmake --build build-vk --target gallery --parallel
```

窗口后端默认是 GLFW。SDL2 不放进 `3rd/`，只有配置 `-DEUI_WINDOW_BACKEND=sdl2` 才参与构建：

- `EUI_DEPS_MODE=auto`：优先查找系统 SDL2 包，找不到时下载固定版本 SDL2。
- `EUI_DEPS_MODE=bundled`：SDL2 后端直接报错，因为没有 vendored SDL2。
- `EUI_DEPS_MODE=fetch`：下载固定版本 SDL2 并参与构建。

## CMake Targets

项目导出 `eui::neo` 静态库 target。顶层构建时 `EUI_BUILD_APPS` 默认开启，会构建 `examples/*.cpp` 下的示例程序；作为其他项目的子目录接入时，`EUI_BUILD_APPS` 默认关闭，只提供库和辅助函数。新增内置 demo 时只需要添加 `examples/<name>.cpp`，同名可执行目标和 release 运行包收集都会自动跟随。

- `eui::neo`：核心 DSL、Runtime、图元、组件可用的库 target。
- `eui_neo_configure_app(target)`：给完整 EUI app 配置平台链接选项并复制 `assets/`。
- `eui_neo_copy_assets(target)`：只复制 EUI-NEO 自带运行资源。

安装包方式额外导出 `EuiNeoConfig.cmake` 和 `EuiNeoTargets.cmake`。安装后外部项目使用：

```cmake
find_package(EuiNeo CONFIG REQUIRED)
target_link_libraries(my_app PRIVATE eui::neo)
```

`add_subdirectory` 仍推荐使用 `eui::neo`，安装包导出名同样为 `eui::neo`，和源码子目录接入保持一致。

## 本地构建

macOS / Linux:

```sh
cmake -S . -B build
cmake --build build
./build/gallery
```

默认使用 GLFW 窗口后端和 OpenGL 渲染后端。OpenGL 开发文件不可用时，安装平台 OpenGL/windowing 开发包，或切到 Vulkan 构建目录：`cmake -S . -B build-vk`。

Windows / PowerShell:

```powershell
cmake -S . -B build
cmake --build build --config Release
.\build\Release\gallery.exe
```

Linux 依赖示例：

```sh
sudo apt-get install -y ninja-build libx11-dev libxrandr-dev libxinerama-dev libxcursor-dev libxi-dev libgl1-mesa-dev libcurl4-openssl-dev
# SDL2 后端可选：
sudo apt-get install -y libsdl2-dev
```

SDL2 后端使用系统包：

```sh
cmake -S . -B build-sdl2
cmake --build build-sdl2 --target gallery
```

SDL2 后端显式 fetch：

```sh
cmake -S . -B build-sdl2 -DEUI_DEPS_MODE=fetch
cmake --build build-sdl2 --target gallery
```

SDL2 需要固定渲染后端时，使用单独构建目录：

```sh
cmake -S . -B build-sdl2-vk
cmake --build build-sdl2-vk --target gallery
```

## 验证命令

常规改动至少验证严格离线构建：

```sh
cmake -S . -B /tmp/eui-bundled -DCMAKE_BUILD_TYPE=Release -DEUI_DEPS_MODE=bundled
cmake --build /tmp/eui-bundled --parallel
```

CMake package / install/export 改动需要额外验证安装包和消费方工程：

```sh
cmake -S . -B /tmp/eui-install-build -DCMAKE_BUILD_TYPE=Release -DEUI_BUILD_APPS=OFF
cmake --build /tmp/eui-install-build --target eui_neo --parallel
cmake --install /tmp/eui-install-build --prefix /tmp/eui-install
cmake -S tests/consumers/find-package -B /tmp/eui-consumer -DCMAKE_PREFIX_PATH=/tmp/eui-install
cmake --build /tmp/eui-consumer --parallel
```

组件和布局改动建议额外打开视觉回归 fixture：

```sh
cmake -S . -B build-md4c-bundled-check -DEUI_DEPS_MODE=bundled -DEUI_BUILD_TEST_FIXTURES=ON
cmake --build build-md4c-bundled-check --target layout_regression_viewer text_alignment_viewer markdown_viewer loader_lifecycle_viewer --parallel
ctest --test-dir build-md4c-bundled-check -L unit --output-on-failure
./build-md4c-bundled-check/layout_regression_viewer
./build-md4c-bundled-check/text_alignment_viewer
./build-md4c-bundled-check/markdown_viewer
./build-md4c-bundled-check/loader_lifecycle_viewer
```

其中 `layout_regression_viewer` 覆盖 `card`、padding、`wrapContentHeight`、`fill`、overlay 背景、flex 边界和 `scrollView` 内容高度；`text_alignment_viewer` 用于检查文本垂直居中和字体基线；`markdown_viewer` 用于检查 Markdown block / inline / table / code 的显示；`loader_lifecycle_viewer` 用于检查 Loader 的 destroy-on-hide 和 keep-alive 实例状态。

Vulkan shader 源变更必须同步生成头并检查：

```sh
python3 scripts/generate_vulkan_shaders.py
cmake --build build-vk --target eui_check_vulkan_shaders
```

MinGW 交叉编译验证建议在本机或 CI 临时构建目录中完成，至少确认 `eui_neo` 静态库可编译；Windows 可执行文件仍以真机运行为准。

依赖链路改动额外验证联网 fetch：

```sh
cmake -S . -B /tmp/eui-fetch -DCMAKE_BUILD_TYPE=Release -DEUI_DEPS_MODE=fetch
cmake --build /tmp/eui-fetch --parallel
```

发布前检查源码包体积：

```sh
git archive --format=zip --output=/tmp/eui-source.zip HEAD
ls -lh /tmp/eui-source.zip
```

## 提交规范

项目提交使用带 scope 的 Conventional Commits：

```text
type(scope): subject
```

允许的 type：

- `feat`：新增功能。
- `fix`：修复问题。
- `docs`：文档改动。
- `style`：格式、空白、代码风格，不改变行为。
- `refactor`：重构，不新增功能也不修复问题。
- `perf`：性能优化。
- `test`：测试相关。
- `build`：构建系统或依赖管理。
- `ci`：CI / release workflow。
- `chore`：维护性改动。
- `revert`：回滚提交。

规则：

- 必须包含 scope，例如 `docs(readme)`、`ci(release)`、`fix(input)`。
- type 和 scope 使用小写英文。
- subject 使用简短英文祈使句，不加句号。
- 避免 `update docs`、`fix bug`、`change files` 这类模糊描述。
- 提交前用 `git log --oneline -5` 确认新提交信息符合规范。

示例：

```text
docs(readme): align dependency setup documentation
ci(release): add macos release packaging
docs(workflow): add development and review guides
fix(input): correct emoji caret positioning
feat(text): add bundled emoji rendering
chore(deps): trim bundled third-party sources
```

## Release 约定

- 推送 `v*` tag 会触发 `.github/workflows/release.yml`。
- CI 构建 Windows、Linux、macOS 运行包和 SDK 包。
- 运行包会从 `examples/*.cpp` 自动推导可执行文件列表；新增 demo 不需要手动维护 release.yml 的应用名单。
- Release assets 上传运行包和 SDK 包，不上传正文截图。
- Release 正文截图使用已提交的仓库图片或 GitHub 正文附件。
- `docs/releases/` 是本地 release 草稿和素材目录，已被 `.gitignore` 忽略，不提交。

## Release 检查

发布前确认：

- `git status --short` 为空。
- `vX.Y.Z` tag 指向预期提交。
- Windows、Linux、macOS workflow 均通过。
- Release assets 只包含运行包和 SDK 包。
- GitHub 自动 Source code 包体积处于合理范围。
- Release note 中的截图能在网页正文显示。
