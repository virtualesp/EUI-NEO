# 图片、网络图片、SVG 与 GIF

当前项目有图片 DSL：`ui.image(id)`。它和 `ui.rect`、`ui.text` 一样由内部 Runtime 管理，不需要页面或组件直接创建后端 primitive。

## DSL 写法

```cpp
ui.image("logo")
    .size(96.0f, 96.0f)
    .source("assets/icon.svg")
    .radius(18.0f)
    .opacity(0.92f)
    .cover()
    .build();

ui.image("remote")
    .size(260.0f, 150.0f)
    .source("https://example.com/picture.png")
    .cover()
    .radius(16.0f)
    .build();

ui.image("bing")
    .size(260.0f, 150.0f)
    .bingDaily(0, "zh-CN")
    .cover()
    .radius(16.0f)
    .build();

ui.image("loader")
    .size(54.0f, 54.0f)
    .source("mona-loading-default.gif")
    .contain()
    .build();

ui.svg("inline.heart")
    .size(54.0f, 54.0f)
    .source(R"svg(<svg viewBox="0 0 24 24" xmlns="http://www.w3.org/2000/svg">
  <path fill="#ffffff" d="M12 21s-8-4.7-8-11a5 5 0 0 1 8-4 5 5 0 0 1 8 4c0 6.3-8 11-8 11z"/>
</svg>)svg")
    .tint({1.0f, 0.36f, 0.58f, 1.0f})
    .contain()
    .build();
```

`components::image(ui, id)` 会返回一个已经套用默认 theme token 的 `ImageBuilder`。如果页面有自己的 light/dark token，推荐传入：

```cpp
components::image(ui, "avatar", tokens)
    .source("assets/icon.png")
    .cover()
    .build();
```

## 图片主题色

需要让按钮、面板、卡片等 UI 颜色跟随图片时，可以直接调用图片 facade：

```cpp
const eui::Color accent = eui::image::themeColor(
    "bing://daily?idx=0&mkt=zh-CN",
    {0.38f, 0.72f, 0.96f, 1.0f});
```

`themeColor(source, fallback)` 会复用底层图片路径解析、远程下载、Bing daily 下载和静态图解码缓存。图片还没下载好、路径无效或解码失败时返回 `fallback`；图片可用后会从 RGBA 像素里采样一个更适合作为 UI 强调色的颜色，并按解析后的文件路径、文件大小和修改时间缓存成功采样的结果，避免每帧重复扫描像素；fallback 不会进入缓存，同一路径文件更新后会重新解码和采样。

如果需要知道网络图片是否仍在下载，可以传入 `pending`：

```cpp
bool pending = false;
const eui::Color accent = eui::image::themeColor(source, fallback, false, &pending);
```

## 支持能力

- 本地 PNG / JPG / BMP / WebP 等位图通过 `3rd/stb_image.h` 解码。
- 本地或网络 GIF 通过 `stb_image` 解码；多帧 GIF 会按文件内 delay 自动播放。
- 本地 SVG 文件通过 `ui.image(...).source(path)` 加载，内联 SVG 字符串通过 `ui.svg(...).source(svgMarkup)` 加载，二者都通过 `3rd/nanosvg.h` 和 `3rd/nanosvgrast.h` 栅格化成 RGBA 纹理。
- Windows 的 `http://` / `https://` 图片使用系统 `URLDownloadToFileA` 下载到临时缓存目录。
- Linux / 其他非 Windows 平台通过 libcurl 请求网络资源；CMake 在非 Windows 平台要求 `CURL::libcurl`。
- 网络图片下载走统一 内部 async 线程池；下载期间不阻塞界面，下载完成后 Runtime 会触发重绘。
- 静态图片纹理按解析后的文件路径缓存；内联 SVG 按元素 key 和 SVG 内容 hash 缓存，同一份内容重复使用不会重复上传纹理。
- GIF 会缓存已解码帧，并在播放时复用当前渲染后端的一张动态纹理更新像素。
- `bingDaily(idx, mkt)` 内置 Bing 每日图：先请求 Bing API，再下载返回的图片 URL。

## GIF 播放

GIF 不需要额外 API，继续使用 `.source(...)`：

```cpp
ui.image("loading")
    .size(48.0f, 48.0f)
    .source("assets/mona-loading-default.gif")
    .contain()
    .build();
```

Runtime 会把多帧 GIF 视为图片动画，自动保持刷新。帧间隔来自 GIF 文件本身；过短 delay 会被限制到最小 `10ms`，避免异常文件触发过高刷新频率。

Gallery 的 Bing 图片卡片使用 `mona-loading-default.gif` 作为图片下载期间的占位动画；Bing 图片加载成功后会覆盖这个占位图。

## Fit 模式

默认是 `ImageFit::Cover`。

```cpp
.cover()    // 铺满区域，保持比例，超出部分裁剪
.contain()  // 完整显示图片，保持比例，区域可能留空
.stretch()  // 拉伸到区域，可能变形
.fit(eui::ImageFit::Cover)
```

Gallery 的 about logo 和 Bing 图片都使用 cover，因此图片不会被压缩变形，而是适应裁剪。

## 可动画属性

Image 进入 dirty rect 流程，支持这些属性动画：

- frame：位置和大小
- tint / color
- opacity
- radius
- transform：translate / scale / rotate / rotateX / rotateY / translateZ / perspective
- GIF 帧动画：由图片文件 delay 驱动，自动刷新当前图片区域

示例：

```cpp
ui.image("avatar")
    .size(active ? 120.0f : 88.0f, active ? 120.0f : 88.0f)
    .source("assets/icon.png")
    .radius(active ? 32.0f : 18.0f)
    .rotateY(active ? 0.48f : 0.0f)
    .translateZ(active ? 48.0f : 0.0f)
    .perspective(420.0f)
    .transition(0.28f, eui::Ease::OutCubic)
    .animate(eui::AnimProperty::Frame |
             eui::AnimProperty::Radius |
             eui::AnimProperty::Transform)
    .build();
```

## 交互

Image 支持通用交互：

```cpp
ui.image("github.logo")
    .size(96.0f, 96.0f)
    .source("assets/icon.png")
    .onClick([] {
        eui::platform::openUrl("https://github.com/sudoevolve/EUI-NEO");
    })
    .build();
```

但 Image 没有内置 hover / pressed 视觉状态。需要 hover 变色、按压缩放等效果时，推荐外层使用 `Stack + Rect + Image` 组合，或封装成组件。

## 网络文本 API 示例

通用网络文本请求通过 `eui/network.h`：

```cpp
eui::network::requestText(
    "bing.meta",
    "https://www.bing.com/HPImageArchive.aspx?format=js&n=1&idx=0&mkt=zh-CN");

const eui::network::TextResult result = eui::network::textResult("bing.meta");
if (result.ready && result.ok) {
    ui.text("api.text")
        .size(520.0f, 72.0f)
        .text(result.body)
        .wrap(true)
        .build();
}
```

`requestText` 是后台请求，不阻塞 UI。请求完成后会通过统一 async 队列回到 UI 主线程，唤醒 GLFW 事件循环，`include/eui/dsl_app.h` 会在收到完成信号后重新 compose。

Gallery 的 Bing 页面包含本地 PNG、本地 SVG、Bing daily 图片和 Bing API 文本示例。`examples/card_slider.cpp` 展示了手机端随机网络图片、最新 slide 卡片组件、古诗词一言 API 和 `eui::image::themeColor(...)` 联动按钮主题色的用法。

## 当前限制

- SVG 当前按固定目标尺寸栅格化，适合图标和中等尺寸图片；还没有按元素尺寸动态重新栅格化。
- GIF 当前会把所有帧解码到内存，适合小型 loading / sticker 动画；不建议直接使用超大尺寸、超长帧数 GIF。
- 网络请求没有暴露 header、body、进度等高级能力；通用后台任务可以直接使用 `app::async`。
- 图片点击可以做，但 hover / pressed 视觉状态需要自己组合 rect。
