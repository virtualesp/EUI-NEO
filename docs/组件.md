# 组件

组件层位于 `components/`。它不是另一套渲染系统，也不直接持有后端 primitive；组件只负责把常用 DSL 结构封装成更短、更稳定的 builder 调用。

页面仍然由 `eui::Ui` 里的 `row`、`column`、`stack`、`rect`、`text`、`image`、`polygon` 等图元组成。组件内部也只声明 DSL 树，事件、hover、pressed、transition、dirty rect 和 framebuffer cache 都继续交给 内部 Runtime。

## API 审计

本轮只整理公开 API 口径，不重构组件实现。`components/components.h` 的 include 顺序按上表导出顺序维护，README、gallery 和本文档应使用同一套推荐名称。

命名结论：

- 推荐名称保留当前 gallery 使用方式：`dataTable`、`linechart`、`barchart`、`piechart`。
- 兼容别名继续保留：`datatable`、`lineChart`、`barChart`、`pieChart`。
- 新文档和 gallery 示例优先使用推荐名称；别名只在“兼容别名”里说明，不再混作主入口。
- `datepicker`、`timepicker`、`colorpicker` 继续使用无下划线全小写命名。

## 基本约定

- 组件 id 必须稳定，内部子节点使用 `id + ".name"`。
- 组件 API 使用无下划线命名，例如 `datepicker`、`timepicker`、`colorpicker`。
- 大多数组件是受控组件：页面传入当前值，组件通过回调返回 next value。
- 组件不保存业务状态，不提供生命周期钩子。
- 需要动画时优先使用组件的 `.transition(...)`，或者组件内部固定的 DSL transition。
- 弹层类组件一般提供 `.zIndex(...)` / `.z(...)`，页面需要保证它们排在普通内容之上。
- 弹层、侧边栏、菜单和浮层需要在遮罩或面板背景上放 interactive blocker，用透明或实体 hit rect 阻断下层 hover / click / focus。
- `.style(...)` 用于完全覆盖样式结构，`.theme(tokens)` 用于按当前主题重建样式。
- 组件内部不应该 `new` primitive、不绕过 Runtime、不自己读取原始鼠标状态。
- `components/workshop/` 用于高度定制、视觉风格强的创意组件；基础容器和通用控件仍放在 `components/` 根目录。

## 基础包装

`panel`、`text`、`label`、`image` 返回的仍然是 core DSL builder，所以可以继续链式调用底层能力。

```cpp
components::panel(ui, "card", tokens)
    .size(360.0f, 220.0f)
    .transition(0.18f)
    .build();

components::text(ui, "title", tokens)
    .size(420.0f, 48.0f)
    .text("EUI Gallery")
    .fontSize(38.0f)
    .horizontalAlign(eui::HorizontalAlign::Center)
    .verticalAlign(eui::VerticalAlign::Center)
    .build();

components::image(ui, "cover", tokens)
    .size(320.0f, 180.0f)
    .source("assets/icon.png")
    .cover()
    .build();
```

`text.h` 还提供 `titleTextStyle(tokens)`、`subtitleTextStyle(tokens)`、`bodyTextStyle(tokens)`，用于快速生成 `eui::TextStyle`。

### card / layoutDebugOverlay

`card` 是基础容器组件，用于统一背景、圆角、边框、阴影和内容 inset。它仍然只声明 `Stack + Rect + content Stack`，不会保存业务状态。内容回调内的坐标从 inset 后的内容区域开始计算。

```cpp
components::card(ui, "settings.card")
    .position(24.0f, 80.0f)
    .size(360.0f, 180.0f)
    .padding(18.0f)
    .theme(tokens)
    .content([&] {
        ui.text("settings.card.title")
            .size(324.0f, 28.0f)
            .text("Settings")
            .fontSize(20.0f)
            .verticalAlign(eui::VerticalAlign::Center)
            .build();
    })
    .build();
```

常用 API：`position`、`x`、`y`、`size`、`width`、`height`、`wrapContentHeight`、`padding` / `inset`、`style`、`theme`、`color`、`radius`、`border`、`shadow`、`opacity`、`zIndex` / `z`、`content`。

`layoutDebugOverlay` 用于视觉定位布局问题。它只根据传入的 `width`、`height`、`inset` 绘制 frame / padding / content 边界，不读取目标组件的实际 frame。调试时需要把它放在和目标区域相同的位置和尺寸上。

```cpp
ui.stack("settings.card.debug.anchor")
    .position(24.0f, 80.0f)
    .size(360.0f, 180.0f)
    .content([&] {
        components::layoutDebugOverlay(
            ui,
            "settings.card.debug",
            360.0f,
            180.0f,
            18.0f,
            "frame / padding / content");
    })
    .build();
```

`tests/fixtures/layout_regression_viewer.cpp` 覆盖 `padding`、`wrapContentHeight`、`fill` 和 `scrollView` 内容高度；`tests/fixtures/text_alignment_viewer.cpp` 用于观察不同字体和文本类型的垂直居中基线。

## 控件

### button

```cpp
components::button(ui, "save")
    .size(180.0f, 54.0f)
    .theme(tokens)
    .icon(0xF0C7)
    .text("Save")
    .transition(0.18f)
    .onClick([] {
        // update page state
    })
    .build();
```

常用 API：`size`、`scale`、`text`、`icon`、`fontSize`、`iconSize`、`textColor`、`iconColor`、`style`、`theme`、`primaryTheme`、`secondaryTheme`、`radius`、`rounding`、`opacity`、`disabled`、`enabled`、`translate`、`translateX`、`translateY`、`pressScale`、`border`、`shadow`、`colors`、`transition`、`onClick`、`onContextMenu`。

按钮内部是 `Stack + Rect + Row + Text`。按压缩放作用在外层 stack，背景、图标和文字会一起动。

### checkbox / radio / toggleSwitch

这些都是受控组件，页面传当前值，回调里写回状态。

```cpp
components::checkbox(ui, "remember")
    .theme(tokens)
    .checked(remember)
    .text("Remember me")
    .onChange([&](bool value) {
        remember = value;
    })
    .build();

components::radio(ui, "mode.a")
    .selected(mode == 0)
    .text("Mode A")
    .onSelect([&] {
        mode = 0;
    })
    .build();

components::toggleSwitch(ui, "motion")
    .checked(enableMotion)
    .label("Motion")
    .onChange([&](bool value) {
        enableMotion = value;
    })
    .build();
```

`checkbox` 支持 `size`、`checked`、`text`、`fontSize`、`boxSize`、`style`、`theme`、`transition`、`onChange`。

`radio` 支持 `size`、`selected` / `checked`、`text`、`fontSize`、`dotSize`、`style`、`theme`、`transition`、`onSelect`、`onChange`。

`toggleSwitch` 支持 `size`、`checked`、`label` / `text`、`fontSize`、`trackSize`、`style`、`theme`、`transition`、`onChange`。

### progress / slider

```cpp
components::progress(ui, "loading")
    .theme(tokens)
    .size(320.0f, 14.0f)
    .value(progress)
    .build();

components::slider(ui, "volume")
    .theme(tokens)
    .size(320.0f, 34.0f)
    .value(volume)
    .onChange([&](float value) {
        volume = value;
    })
    .build();
```

`progress` 支持 `size`、`value`、`style`、`theme`、`transition`。

`slider` 支持 `size`、`value`、`style`、`theme`、`transition`、`onChange`。`value` 会 clamp 到 `0.0f - 1.0f`。滑块用透明 hit rect 处理点击和拖拽，拖拽时根据缓存 bounds 计算 next value。

### markdown

`markdown` 是基于 MD4C 的显示组件。它不进入 core primitive，也不执行 HTML/CSS；组件把 Markdown 解析结果组合成现有 DSL 图元。组件内部会按 source、宽度和文本样式缓存解析结果与行内排版结果，避免稳定 Markdown 在反复 compose 时重复解析和测量；缓存归属当前 `ui.state` 生命周期，不改变显示结果。

```cpp
components::markdown(ui, "doc")
    .theme(tokens)
    .width(720.0f)
    .markdown(markdownText)
    .wrapContentHeight()
    .build();
```

常用 API：`markdown` / `text` / `source`、`width`、`size`、`height`、`wrapContentHeight`、`margin`、`position`、`theme`、`style`、`zIndex`。

当前接入 MD4C 的 block/span/text 回调面：标题、段落、引用、无序/有序/任务列表、分隔线、fenced/indented code、HTML block、表格、表格对齐、emphasis、strong、link/autolink、image、inline code、strikethrough、LaTeX math、wiki link、underline、inline HTML、entity、soft/hard break 等。inline code、link/autolink、image、LaTeX math、wiki link、inline HTML、删除线和下划线会在组件内部 line-box 排版中映射为可见文本、胶囊或装饰线；HTML/CSS 不执行，图片不会加载真实资源，而是显示为占位文本。代码、引用、HTML block 和表格使用块级样式显示。

### mouseArea

`mouseArea` 是通用输入热区，不负责业务状态，也不画默认 UI。它适合给 canvas、图片、卡片、carousel、时间轴这类区域补一层统一的鼠标/触摸板输入 API。

```cpp
components::mouseArea(ui, "stage.input")
    .size(width, height)
    .scrollStep(0.18f)
    .maxScrollStep(1.0f)
    .dragThreshold(3.0f)
    .onTap([](const components::MouseEvent& event) {
        // event.x / event.y 是热区内逻辑坐标
    })
    .onHover([](bool hover) {
        // hover changed
    })
    .onMove([](const components::MouseEvent& event) {
        // hover 时持续返回热区内局部坐标，可用于 pointer tilt / spotlight
    })
    .onDragStart([](const components::MouseEvent& event) {
        // drag threshold crossed
    })
    .onDrag([](const components::MouseDragEvent& event) {
        // event.totalX / totalY 是从按下开始的逻辑位移
    })
    .onScroll([](const components::MouseScrollEvent& event) {
        // event.stepY 已按 scrollStep 和 maxScrollStep 归一
    })
    .onContextMenu([](const components::MouseEvent& event) {
        // right click
    })
    .build();
```

常用 API：`position`、`size`、`zIndex` / `z`、`radius` / `rounding`、`color`、`cursor`、`disabled` / `enabled`、`scrollStep`、`maxScrollStep`、`dragThreshold`、`suppressClickAfterDrag`、`onTap` / `onClick`、`onPress`、`onRelease`、`onHover` / `onHoverChanged`、`onEnter`、`onLeave`、`onMove`、`onDragStart`、`onDrag`、`onDragEnd`、`onScroll` / `onWheel`、`onContextMenu`。

`MouseEvent` 提供局部坐标 `x/y`、全局逻辑坐标 `globalX/globalY`、本帧位移 `deltaX/deltaY`、热区 bounds 和按键状态。`MouseDragEvent` 额外提供 `totalX/totalY`。`MouseScrollEvent` 提供原始滚轮 `deltaX/deltaY` 和处理后的 `stepX/stepY`。

### input

```cpp
components::input(ui, "name")
    .theme(tokens)
    .size(280.0f, 42.0f)
    .value(name)
    .placeholder("Hello EUI-NEO 😉")
    .fontFamily("monospace")
    .onChange([&](const std::string& value) {
        name = value;
    })
    .onEnter([] {
        // submit
    })
    .build();
```

支持 `size`、`text` / `value`、`placeholder`、`multiline`、`fontSize`、`fontFamily`、`inset`、`style`、`theme`、`transition`、`onChange`、`onEnter`、`onFocus`。

当前 input 支持 focus、点击定位光标、拖动选择、Shift+方向键选择、Ctrl+A/C/X/V/Z、Cmd+A/C/X/V/Z、Ctrl+Y、Ctrl+Shift+Z、Cmd+Shift+Z、Backspace、Delete、Enter、单行和多行模式，并会把系统 IME 候选框锚定到当前光标位置。光标、选择、删除、撤销、重做和横向滚动基于 `TextPrimitive::measureTextMetrics(...)` 的 shaped caret stops；这些 stops 使用 UTF-8 byte offset，并与实际渲染的 fallback、emoji 缩放和 glyph advance 保持一致，适配 emoji、CJK、阿拉伯连字和组合符号。多行模式会按完整内容高度声明内部文本 frame，再由外层 input clip 裁剪，因此滚动到较多行时不会因为子元素 frame 离开 viewport 而被 Runtime 跳过绘制。默认字体族是跨平台 `monospace`，默认占位文本是 `Hello EUI-NEO 😉`。暂时没有 IME 组合态显示。

### scroll

`scroll` 只绘制滚动条，并把 track/thumb 绑定到 Runtime 滚动状态。它不直接持有内容；内容裁剪和位移应交给同一个 Runtime scroll state，通常由 `scrollView` 统一创建。

```cpp
ui.stack("list.viewport")
    .size(360.0f, 260.0f)
    .clip()
    .scrollState("list.viewport", scrollY, contentHeight - 260.0f, 48.0f)
    .onScrollOffsetChanged([&](float value) {
        scrollY = value;
    })
    .content([&] {
        ui.column("list.content")
            .size(360.0f, contentHeight)
            .scrollContentFrom("list.viewport")
            .build();
    })
    .build();

components::scroll(ui, "list.scroll")
    .state("list.viewport")
    .size(8.0f, 260.0f)
    .viewport(260.0f)
    .content(contentHeight)
    .offset(scrollY)
    .build();
```

支持 `x`、`y`、`position`、`size`、`state`、`offset` / `value`、`viewport` / `viewportHeight`、`content` / `contentHeight`、`step`、`zIndex` / `z`、`style`、`theme`、`transition`。

### scrollView

`scrollView` 是在 `scroll` 之上的更高层封装：它负责创建 viewport、自动测量内容高度、在需要时显示滚动条，并把当前 offset 回传给页面状态。滚轮和拖动滚动条时，Runtime 只更新滚动 transform 和 viewport dirty rect，不重新 compose 整个页面。

```cpp
components::scrollView(ui, "page.body")
    .theme(tokens)
    .size(640.0f, 420.0f)
    .offset(scrollY)
    .gap(18.0f)
    .step(48.0f)
    .onChange([&](float value) {
        scrollY = value;
    })
    .content([&](eui::Ui& contentUi, float contentWidth, float viewportHeight) {
        contentUi.column("page.sections")
            .width(contentWidth)
            .height(eui::SizeValue::wrapContent())
            .gap(18.0f)
            .content([&] {
                // compose scroll content here
            })
            .build();
    })
    .build();
```

支持 `size`、`offset` / `value`、`gap` / `spacing`、`step`、`scrollbarWidth`、`scrollbarGap`、`zIndex` / `z`、`style`、`theme`、`transition`、`contentKey` / `measureKey`、`onChange`、`content(...)`。

使用建议：

- 需要“滚动区域自动更新，不想手动维护内容高度”时，优先用 `scrollView`。
- `scrollView` 会先按内容宽度测量；若出现滚动条，再按扣除滚动条后的宽度重测一次，避免文本换行和内容高度不一致。
- 内容高度稳定、但 compose 成本较高时，可以传 `contentKey(...)` 复用测量高度；key 必须包含会改变内容高度的状态，例如主题密度、展开项、数据版本或筛选条件。未传 key 时保持每次自动测量。
- 如果内容测量要完全自定义，可以使用底层 `scroll`，但滚动内容和滚动条必须绑定同一个 Runtime scroll state。

### datepicker / timepicker / colorpicker

这三个 picker 都是受控组件，命名不带下划线。页面传入当前值和 `open`，组件通过 `onChange` / `onOpenChange` 回写 next state。

picker 现在和 `dialog` 一样绘制全屏 overlay + 居中面板，适合放在页面根级 compose 末尾，避免被滚动容器或 `.clip()` 裁剪。触发入口可以用 `button` 或自定义字段单独绘制。

面板打开后会维护组件内部 draft：日期/时间轮盘、滚轮、颜色 slider、预设色块都只更新面板内预览，不会立即改变页面状态。点击 `Done` 才触发 `onChange` 提交 draft；点击遮罩只关闭，不提交。

`datepicker` 支持 `open`、`screen`、`size(panelWidth, panelHeight)`、`date(year, month, day)`、`style`、`theme`、`transition`、`zIndex` / `z`、`onChange`、`onOpenChange`。年份范围是 `1900 - 2200`，月份和日期会按闰年和当月天数修正，`onChange` 只在 `Done` 时触发。

## 当前限制

- DSL 已提供 `ui.loader(...).destroyOnHide()` / `.keepAlive()` 和 `ui.state<T>(id)` 作为实例状态基础；组件私有交互态应接入 `ui.state`，由当前页面和 Loader scope 管理生命周期。
- `scroll` 只负责 offset 控制，不是完整 ScrollArea；需要自动测量内容时应优先用 `scrollView`。
- `input` 暂时没有 IME 组合态显示。
- chart 是轻量展示组件，当前只显示归一化百分比，不提供坐标轴刻度配置、图例布局和数据格式化回调。
- `dialog`、`toast`、`contextMenu`、`dropdown`、`datepicker`、`timepicker`、`colorpicker` 需要页面传入 open/visible 状态，组件不会自己持久化开关状态。
- 容器 transform 会继承到组件内部子树渲染，包含 2D transform 和 2.5D 透视投影；默认 hit-test 仍按布局矩形计算，组件确实需要跟随视觉变换时可在内部元素上开启 `.transformedHitTest()`。
- Runtime 按绘制顺序从最上层往下命中 interactive 元素，同一位置只会触发最上层交互层；focus 命中会被上层 interactive blocker 阻断。
- 复杂嵌套滚动、圆角 clip、事件冒泡等能力仍属于 Runtime 后续增强范围。

## 写新组件时的底线

```cpp
class CounterButtonBuilder {
    struct CounterButtonState {
        int clicks = 0;
    };

public:
    CounterButtonBuilder(eui::Ui& ui, std::string id)
        : ui_(ui), id_(std::move(id)) {}

    CounterButtonBuilder& size(float width, float height) {
        width_ = width;
        height_ = height;
        return *this;
    }

    CounterButtonBuilder& text(std::string value) {
        text_ = std::move(value);
        return *this;
    }

    CounterButtonBuilder& onClick(std::function<void(int)> callback) {
        onClick_ = std::move(callback);
        return *this;
    }

    void build() {
        CounterButtonState& state = ui_.state<CounterButtonState>(id_ + ".state");
        const float w = std::max(1.0f, width_);
        const float h = std::max(1.0f, height_);

        ui_.stack(id_)
            .size(w, h)
            .visualStateFrom(id_ + ".hit", 0.965f)
            .content([&] {
                ui_.rect(id_ + ".hit")
                    .size(w, h)
                    .states(normal_, hover_, pressed_)
                    .radius(std::min(12.0f, h * 0.5f))
                    .onClick([&state, callback = onClick_] {
                        ++state.clicks;
                        if (callback) {
                            callback(state.clicks);
                        }
                    })
                    .build();

                ui_.text(id_ + ".label")
                    .size(w, h)
                    .text(text_ + " " + std::to_string(state.clicks))
                    .fontSize(15.0f)
                    .lineHeight(19.0f)
                    .color({0.94f, 0.96f, 1.0f, 1.0f})
                    .horizontalAlign(eui::HorizontalAlign::Center)
                    .verticalAlign(eui::VerticalAlign::Center)
                    .build();
            })
            .build();
    }

private:
    eui::Ui& ui_;
    std::string id_;
    std::string text_ = "Clicks";
    std::function<void(int)> onClick_;
    float width_ = 160.0f;
    float height_ = 42.0f;
    eui::Color normal_{0.12f, 0.14f, 0.18f, 1.0f};
    eui::Color hover_{0.16f, 0.18f, 0.24f, 1.0f};
    eui::Color pressed_{0.10f, 0.12f, 0.16f, 1.0f};
};

inline CounterButtonBuilder counterButton(eui::Ui& ui, const std::string& id) {
    return CounterButtonBuilder(ui, id);
}
```

使用方式：

```cpp
counterButton(ui, "demo.counter")
    .size(180.0f, 44.0f)
    .text("Counter")
    .onClick([](int clicks) {
        // 业务层只接收事件，不负责释放组件内部 click state。
    })
    .build();
```

重点：

- 组件只是 DSL 组合。视觉、交互和动画都声明到 DSL 树上，由 Runtime 统一执行。
- 组件内部交互态使用 `ui.state<T>(id)`，不要写组件级 `static unordered_map`。
- 页面业务状态仍由页面持有，例如 `open`、`selected`、`checked`、输入文本和数据源。
- 组件 id 必须稳定，内部子节点使用 `id_ + ".name"`，避免和同级组件互相污染。
- 组件放进 Loader 后不需要自己处理生命周期：`destroyOnHide` 会释放 `ui.state`，`keepAlive` 会保留。
