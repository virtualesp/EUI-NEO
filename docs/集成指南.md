# 集成指南

本文面向想把 EUI-NEO 接入自己 C++ 项目的用户。当前推荐四种方式，按易用程度排序：

- 公共头文件方式：用 `include/eui_neo.h` + 框架提供的 app main 快速创建完整桌面 app。
- 静态库方式：链接 `eui::neo`，接入已有 main、窗口或渲染循环。
- 安装包方式：先 `cmake --install`，再在外部项目中 `find_package(EuiNeo CONFIG REQUIRED)`。
- examples 开发方式：直接在 `examples/` 下新增应用，适合快速验证和内置示例。

EUI-NEO 通过 CMake 导出一个库 target：

```cmake
eui::neo
```

当 EUI-NEO 作为子目录接入时，默认不会构建仓库自带的 `gallery`、`eui_demo`、`serial_tool` 等示例程序。需要显式构建示例时，配置 `-DEUI_BUILD_APPS=ON`。

## 快速开始：接入一个最小 CMake 项目

如果你要在自己的项目里直接创建一个完整 EUI-NEO 桌面应用，最小结构如下：

```text
MyProject/
  CMakeLists.txt
  app.cpp
  external/
    EUI-NEO/
```

`CMakeLists.txt`：

```cmake
cmake_minimum_required(VERSION 3.14)
project(MyProject LANGUAGES C CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_subdirectory(external/EUI-NEO)

add_executable(my_app
    external/EUI-NEO/core/app/glfw_app_main.cpp
    app.cpp
)

eui_neo_configure_app(my_app)
```

`app.cpp`：

```cpp
#include "eui_neo.h"

namespace app {

const DslAppConfig& dslAppConfig() {
    static const DslAppConfig config = DslAppConfig{}
        .title("My App")
        .pageId("my_app")
        .windowSize(960, 640)
        .fps(90.0);
    return config;
}

void compose(eui::Ui& ui, const eui::Screen& screen) {
    ui.column("root")
        .size(screen.width, screen.height)
        .padding(32.0f)
        .spacing(12.0f)
        .content([&] {
            ui.text("title")
                .text("Hello EUI-NEO")
                .fontSize(28.0f)
                .lineHeight(34.0f)
                .build();

            components::button(ui, "primary")
                .text("Start")
                .onClick([] {
                    // update app state here
                })
                .build();
        })
        .build();
}

} // namespace app
```

构建：

```sh
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel
./build/my_app
```

Windows 多配置生成器通常这样运行：

```powershell
cmake -S . -B build
cmake --build build --config Release
.\build\Release\my_app.exe
```

这条路径默认使用 GLFW 窗口后端和 OpenGL 渲染后端。需要 Vulkan 时使用独立构建目录：

```sh
cmake -S . -B build-vk -DCMAKE_BUILD_TYPE=Release
```

需要 SDL2 窗口后端时，先把 app main 换成 `external/EUI-NEO/core/app/sdl2_app_main.cpp`，再配置：

```sh
cmake -S . -B build-sdl2 -DCMAKE_BUILD_TYPE=Release
cmake --build build-sdl2 --parallel
```

`eui_neo_configure_app(my_app)` 会完成三件事：链接 `eui::neo`、设置平台相关链接选项、把 EUI-NEO 的 `assets/` 复制到可执行文件目录。

## 1. 公共头文件方式

这是最简单的完整 app 接入方式。它不是纯 header-only：应用代码只包含公共 facade 头文件，构建仍然链接 EUI-NEO 的静态库并使用框架提供的 app main。

推荐先按上面的“快速开始”结构接入：

- `add_subdirectory(external/EUI-NEO)` 把 EUI-NEO 加到你的 CMake 构建里。
- `external/EUI-NEO/core/app/glfw_app_main.cpp` 提供默认 GLFW app main。
- `eui_neo_configure_app(my_app)` 链接 `eui::neo`、设置平台链接选项，并复制 `assets/` 到可执行文件目录。
- 应用只需要实现 `app::dslAppConfig()` 和 `app::compose()`，然后在 `compose()` 里用 DSL 和 components 写界面。

默认窗口后端是 GLFW。需要 SDL2 完整 app 时，配置 `-DEUI_WINDOW_BACKEND=sdl2`，并把 app main 换成 SDL2 版本：

```cmake
add_executable(my_app
    external/EUI-NEO/core/app/sdl2_app_main.cpp
    app.cpp
)
eui_neo_configure_app(my_app)
```

在你自己的项目里直接配置当前项目即可：

```sh
cmake -S . -B build-sdl2 -DCMAKE_BUILD_TYPE=Release
cmake --build build-sdl2 --parallel
```

这样默认字体、图标、图片和示例资源可以按 `assets/...` 路径找到。

## 2. 静态库方式

如果你已经有自己的 main、窗口系统、渲染 context 或事件循环，直接链接 EUI-NEO 导出的静态库 target：

```cmake
add_subdirectory(external/EUI-NEO)

add_executable(my_app main.cpp app.cpp)
target_link_libraries(my_app PRIVATE eui::neo)
eui_neo_copy_assets(my_app)
```

普通 UI 代码仍然优先包含公共 facade：

```cpp
#include "eui_neo.h"
```

只有接入边界需要包含底层 runtime / platform 头文件。比如嵌入已有 GLFW / OpenGL 主循环时，才会直接使用 `core::dsl::Runtime`。

也可以用 CMake `FetchContent` 拉取 EUI-NEO：

```cmake
include(FetchContent)

FetchContent_Declare(
    eui_neo
    GIT_REPOSITORY https://github.com/sudoevolve/EUI-NEO.git
    GIT_TAG        main
)
FetchContent_MakeAvailable(eui_neo)

add_executable(my_app main.cpp app.cpp)
target_link_libraries(my_app PRIVATE eui::neo)
eui_neo_copy_assets(my_app)
```

实际项目建议把 `GIT_TAG` 固定到 release tag 或具体 commit，避免上游变化影响构建复现。

## 3. 安装包 / find_package 方式

从源码安装：

```sh
cmake -S . -B build-install -DCMAKE_BUILD_TYPE=Release -DEUI_BUILD_APPS=OFF
cmake --build build-install --target eui_neo --parallel
cmake --install build-install --prefix /tmp/eui-install
```

外部项目：

```cmake
cmake_minimum_required(VERSION 3.14)
project(MyProject LANGUAGES CXX)

find_package(EuiNeo CONFIG REQUIRED)

add_executable(my_app main.cpp)
target_link_libraries(my_app PRIVATE eui::neo)
```

配置外部项目：

```sh
cmake -S . -B build -DCMAKE_PREFIX_PATH=/tmp/eui-install
cmake --build build --parallel
```

安装包方式适合把 EUI-NEO 作为系统或 CI 缓存中的独立依赖。当前 package 会导出 `eui::neo` target 和公共头文件；默认 GLFW/OpenGL 路径会继续通过 CMake 依赖 target 传播。SDL2 / Vulkan 仍需要消费方环境提供对应 SDK 或包。

## 4. 直接在 `examples/` 开发

快速实验、复现问题或新增内置 demo 时，可以直接在仓库的 `examples/` 下开发：

```text
examples/
  gallery.cpp
  eui_demo.cpp
  serial_tool.cpp
  my_app.cpp
```

`examples/my_app.cpp`：

```cpp
#include "eui_neo.h"

namespace app {

const DslAppConfig& dslAppConfig() {
    static const DslAppConfig config = DslAppConfig{}
        .title("My EUI App")
        .pageId("my_app")
        .windowSize(900, 600)
        .fps(90.0);
    return config;
}

void compose(eui::Ui& ui, const eui::Screen& screen) {
    ui.column("root")
        .size(screen.width, screen.height)
        .padding(32.0f)
        .spacing(12.0f)
        .content([&] {
            ui.text("title")
                .text("Hello EUI-NEO")
                .fontSize(28.0f)
                .lineHeight(36.0f)
                .build();

            components::button(ui, "button")
                .text("Click")
                .onClick([] {
                    // update your app state here
                })
                .build();
        })
        .build();
}

} // namespace app
```

顶层构建会自动扫描 `examples/*.cpp`，并为每个文件生成同名可执行程序：

```sh
cmake -S . -B build
cmake --build build
./build/my_app
```

复杂示例建议参考 `gallery` 的组织方式：`examples/gallery.cpp` 只保留应用入口、全局主题、导航壳和页面路由；具体页面拆到 `examples/pages/*.h`。每个页面文件定义一个页面对象，例如 `GalleryControlsPage`，对象内部持有自己的交互状态，并提供 `compose(eui::Ui&, float width, float height)` 声明页面内容。这种方式接近 QML 的页面组件模型：入口负责装配和切换，页面自己管理自己的状态和画面。

```text
examples/
  gallery.cpp              # app shell, route, shared theme
  pages/
    gallery_controls.h     # struct GalleryControlsPage { state + compose(...) }
    gallery_bing.h
    gallery_about.h
```

`examples/pages/*.h` 是被 `gallery.cpp` 包含的页面组件头，不是独立编译单元。页面里可以直接使用 `gallery.cpp` 中的主题 token、动画 helper 和共享小工具；业务交互状态优先放在页面对象里，而不是继续堆在 `gallery.cpp` 的匿名 namespace。

因此使用 VSCode 的 Microsoft C/C++ 扩展单独打开 `examples/pages/*.h` 时，可能会因为看不到 `gallery.cpp` 中的主题 token、动画 helper 或共享函数而出现红线。这是 IntelliSense 上下文限制，不代表真实构建失败。建议以 CMake 构建结果为准；如果希望减少这类误报，可以改用 clangd，或直接忽略 `examples/pages` 下的 Microsoft C/C++ 红线。

测试 SDL2 后端时，单独开一个构建目录：

```sh
cmake -S . -B build-sdl2
cmake --build build-sdl2 --target serial_tool
./build-sdl2/serial_tool
```

## 嵌入已有 GLFW / OpenGL 主循环

如果你的程序已经自己创建了 GLFW 窗口和 OpenGL context，可以直接持有一个 `core::dsl::Runtime`：

```cpp
#include "core/dsl_runtime.h"
#include "core/input/input_state.h"

core::dsl::Runtime runtime;
runtime.initialize(window);

while (!glfwWindowShouldClose(window)) {
    glfwPollEvents();

    int framebufferWidth = 0;
    int framebufferHeight = 0;
    glfwGetFramebufferSize(window, &framebufferWidth, &framebufferHeight);

    float contentScaleX = 1.0f;
    float contentScaleY = 1.0f;
    glfwGetWindowContentScale(window, &contentScaleX, &contentScaleY);
    const float dpiScale = (contentScaleX + contentScaleY) * 0.5f;
    const float logicalWidth = static_cast<float>(framebufferWidth) / dpiScale;
    const float logicalHeight = static_cast<float>(framebufferHeight) / dpiScale;

    runtime.compose("embedded", logicalWidth, logicalHeight, [](eui::Ui& ui, const eui::Screen& screen) {
        ui.row("root")
            .size(screen.width, screen.height)
            .content([&] {
                ui.text("hello")
                    .text("Embedded EUI-NEO")
                    .position(24.0f, 24.0f)
                    .fontSize(22.0f)
                    .build();
            })
            .build();
    });

    const bool shouldRender = runtime.update(window, deltaSeconds, dpiScale, dpiScale, true);
    if (shouldRender) {
        runtime.render(framebufferWidth, framebufferHeight, dpiScale);
        glfwSwapBuffers(window);
    }
}

runtime.shutdown();
```

嵌入模式下，你需要自己负责：

- `glfwInit()` / `glfwTerminate()`。
- 创建 OpenGL 3.3 core profile context。
- 调用 `gladLoadGLLoader(...)`。
- 安装或转发输入事件。直接使用 EUI-NEO 的 `core/input/input_state.h` 时，可调用 `core::installInputCallbacks(window)`。
- 主循环节流、清屏、swap buffers 和窗口生命周期。

如果不需要完全自定义主循环，优先使用“公共头文件方式”。

## 依赖和资源

EUI-NEO 的第三方依赖由 `3rd/dependencies.cmake` 处理：

- `EUI_DEPS_MODE=auto`：默认，优先用本地 `3rd/`，缺失时联网拉取。
- `EUI_DEPS_MODE=bundled`：严格离线，只使用 `3rd/`。
- `EUI_DEPS_MODE=fetch`：强制联网拉取。

窗口后端默认是 GLFW。SDL2 是可选后端，不 vendored 到 `3rd/`：

- `cmake -S . -B build-sdl2`：优先使用系统 SDL2 包，找不到时下载固定版本 SDL2。
- `cmake -S . -B build-sdl2 -DEUI_DEPS_MODE=fetch`：下载固定版本 SDL2。
- `cmake -S . -B build-sdl2 -DEUI_DEPS_MODE=bundled`：直接报错。

内置和 fetch 下载的依赖默认按静态链接构建，包括 GLFW。Release 包不需要额外携带 GLFW DLL / dylib / so。只有使用系统 SDL2 包时，SDL2 仍可能按系统包提供的方式动态链接。

HarfBuzz 默认开启，可通过 `-DEUI_ENABLE_HARFBUZZ=OFF` 关闭。

MD4C 默认开启，用于 `components::markdown(ui, id)` 解析 Markdown 文本。仓库默认 vendored 在 `3rd/md4c`，可通过 `-DEUI_ENABLE_MARKDOWN=OFF` 关闭；关闭后 Markdown 组件会退化为纯文本段落显示。

Markdown 组件会接入 MD4C 的 block/span/text 回调面，并用现有 DSL primitive 可视化 CommonMark 与启用的 MD4C 扩展：标题、段落、引用、列表、任务列表、分隔线、代码块、HTML block、表格、表格对齐、强调、加粗、链接/自动链接、图片、inline code、删除线、LaTeX math、wiki link、underline、inline HTML、entity、soft/hard break 等。当前组件不是浏览器内核：HTML/CSS 不会执行；inline code、链接、图片、math、wiki、HTML、删除线和下划线会在组件内部 line-box 排版中映射成可见文本、胶囊或装饰线，图片显示为占位文本。代码、引用、HTML block 和表格使用块级样式显示。

运行时资源默认按 `assets/...` 查找。外部项目可以：

- 调用 `eui_neo_copy_assets(my_app)` 复制 EUI-NEO 自带资源。
- 在自己的构建系统中复制资源。
- 通过 `DslAppConfig::textFont(...)`、`iconFont(...)`、`iconPath(...)` 指向自己的资源。

## 适合公开给用户的最小承诺

- C++17。
- CMake 3.14+。
- OpenGL 3.3 core profile。
- GLFW 窗口输入。
- `eui::neo` 是库 target；示例程序是可选目标。
